# Computer-Use AI 작업 규칙

이 폴더는 Windows 사용자가 ZIP으로 전달받아 Codex CLI, Claude Code, Gemini CLI 같은 AI 작업 도구에서 바로 사용할 수 있는 실행 패키지다. 사용자는 기능 이름이나 내부 명령을 몰라도 자연어로 목표만 말할 수 있어야 한다.

## 모든 작업의 시작

1. 현재 작업 폴더를 이 파일이 있는 프로젝트 루트로 고정한다. 일부 파일만 다른 곳으로 복사하지 않는다.
2. `node scripts/ai_project_check.mjs --json`을 실행해 패키지 완전성, Windows 연결 가능 여부, 기능별 진입점을 확인한다.
3. `docs/ai_agent_guide.md`에서 요청과 관련된 절만 읽고 기존 스크립트를 우선 사용한다.
4. 실행 전 카카오톡·위챗 로그인, 대상 창, 웹 로그인처럼 자동으로 해결할 수 없는 사용자 상태만 확인한다.
5. 작업을 실행하고 산출물의 manifest, audit, 종료 코드로 검증한다. 단순히 창이 열렸거나 명령이 시작됐다는 이유로 완료라고 말하지 않는다.

Node.js를 찾지 못하면 Windows에서 `1_백업_시작.bat`를 실행해 기본 도구 설치를 먼저 시도한다. WSL은 선택 사항이며, Windows GUI 작업은 WSL 세션에서도 `powershell.exe`와 Windows 앱으로 실행한다.

## 기능 선택

- 현재 열린 카카오톡 또는 위챗 방, 왼쪽 목록 전체 백업: `docs/ai_agent_guide.md`의 채팅 백업 절
- 백업 결과 열기, 품질 확인, 준비·검증 보고서: 결과 및 진단 절
- 웹사이트 읽기, 입력, 선택, 업로드, 제출: Chrome 자동화 절과 `docs/browser_workflow_playbook.md`
- 이 패키지에 없는 새 B/C/D 기능 구현: 먼저 지원 범위를 설명하고, 기존 Windows-first 구조와 검증 계약을 확장한다.

## 실행 원칙

- 대상 환경은 Windows 10/11이다. 네이티브 Windows PowerShell 경로를 기본으로 사용하고 WSL 전용 명령을 필수로 만들지 않는다.
- 카카오톡·위챗과 Chrome 화면 작업은 전경 창을 사용한다. 두 GUI 작업을 동시에 실행하지 않는다.
- 사용자가 백업이나 웹 제출을 명확히 요청했다면 그 요청을 실행 의사로 취급한다. 로그인, MFA, CAPTCHA, 결제, 삭제, 대상이 모호한 제출처럼 사람이 판단해야 하는 지점에서만 멈춘다.
- CAPTCHA나 MFA를 우회하지 않는다. 로그인된 세션과 사용자가 직접 완료한 인증을 이어서 사용한다.
- 채팅 본문, 스크린샷, 번역 대상은 개인정보일 수 있다. 사용자가 개인정보 외부 전송에 명시적으로 동의하기 전에는 `--allow-cloud-ocr`, `--allow-cloud-translation`을 사용하거나 외부 AI로 보내지 않는다.
- 일반 Chrome 프로필을 디버그 모드로 다시 열거나 종료하지 않는다. 프로젝트의 전용 Chrome 프로필과 자동 선택된 로컬 CDP 포트를 사용한다.
- 로컬 웹 서버를 외부 인터페이스에 바인딩하지 않는다. 기본 `127.0.0.1` 범위를 유지한다.
- 사용자가 요청하지 않은 파일 삭제, 기존 백업 덮어쓰기, 앱 설정 변경을 하지 않는다.

## 완료 기준

- 채팅 백업: 생성된 방 수, 실패·확인 필요 수, 목록 끝 도달 여부, 방별 이력 상한을 manifest와 audit에서 확인한다.
- 웹 자동화: 정확한 탭을 선택하고, 실행 후 `browser_workflow.mjs audit ... --check`가 통과하며 필요한 화면 증거가 있어야 한다.
- 진단: `doctor.mjs --json`의 기본 필수 항목과 요청 기능의 요구사항을 구분한다. 선택 기능의 경고를 전체 실패로 오해하지 않는다.
- 완료 보고에는 수행한 작업, 검증 결과, 산출물 위치, 남은 수동 확인만 간결하게 적는다. 읽지 않은 데이터나 확인하지 않은 전체 범위를 추정해 성공이라고 말하지 않는다.

## 패키지 경계

공개 ZIP이 기본 제공하는 범위는 카카오톡·위챗 화면 백업, 결과 조회·검수, 보이는 Windows Chrome 웹 자동화다. 임의의 모든 Windows 앱 자동화, 메신저 내부 DB 복호화, 원본 첨부파일 일괄 다운로드, CAPTCHA 우회, 인터넷 공개용 서버, MCP 서버 자체는 현재 완성 기능이 아니다. 관련 요청은 기존 기능으로 가능한 부분과 새 구현이 필요한 부분을 분리해 사용자에게 알린다.

외부의 `WINDOWS-CHROME-CDP-자동화-가이드.md`나 개인 PC 경로는 필요하지 않다. 이 폴더의 문서와 스크립트만 사용한다.
