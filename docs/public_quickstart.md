# 카카오톡/위챗 백업 빠른 시작

Windows에서 카카오톡과 위챗 대화를 내 컴퓨터에 백업하고 결과를 확인하는 도구입니다. 처음 목표는 방 하나를 백업하고 `결과 보기`에서 확인하는 것입니다. 앱에서 백업할 방을 선택하고 앱 창을 앞에 둔 뒤 웹 화면에서 백업을 시작합니다. 백업이 시작되면 끝날 때까지 앱 창을 최소화하거나 다른 창으로 가리지 않습니다. 처음 쓰는 사람은 두 개만 보면 됩니다. 안내는 압축을 푼 폴더의 `00_처음_여기부터.txt`, 실행은 `1_백업_시작.bat`입니다. Windows가 파일 확장자를 숨기면 `1_백업_시작`으로 보일 수 있습니다. `시작하기.bat`는 같은 화면을 여는 보조 이름입니다.

기본 사용은 더블클릭과 웹 화면 버튼만으로 진행합니다. 예전 백업 파일 검사나 추가 검수 같은 특별한 기능은 처음에는 건너뛰어도 됩니다. 첫 카카오톡/위챗 백업은 Windows 앱과 문자 인식만 준비하면 시작할 수 있습니다. 기본 버튼으로 실행한 백업은 이 컴퓨터의 백업 폴더에 저장되고, 외부 검수/번역 기능은 기본으로 꺼져 있습니다. `백업 저장 폴더를 준비하지 못했습니다` 또는 저장할 수 없다는 안내가 나오면 압축을 푼 폴더를 바탕화면이나 문서처럼 내가 저장할 수 있는 일반 폴더로 옮긴 뒤 `1_백업_시작.bat`를 다시 실행합니다.
여러 방을 한 번에 저장하려면 첫 화면의 `위챗 통째 백업 확인` 또는 `카카오톡 통째 백업 확인`을 누릅니다. 이 흐름이 백업 화면의 `여러 방 저장` 영역입니다. 누른 뒤 앱 창을 앞에 두고 왼쪽 채팅 목록이 보이게 맞춘 다음 `전체 목록 확인 준비 완료` 체크박스를 선택하고 `전체 목록 확인`을 누릅니다. 후보가 맞을 때 `목록 백업 실행`으로 이어집니다. 첫 화면에도 `전체 목록 확인` -> `후보 확인` -> `목록 백업 실행` -> `결과 보기와 전체 저장` 순서가 보입니다. 완료 카드에 `왼쪽 목록 끝까지 백업했습니다`가 보이면 새 방이 더 이상 나오지 않아 목록 끝까지 확인한 상태입니다. 전체 목록 확인 중 `상한 늘려 전체 목록 다시 확인`이 보이면 아직 방을 클릭하지 않은 상태이므로 그 버튼으로 후보 확인부터 다시 진행합니다. 앱이나 문자 인식이 막혀 있으면 화면에 보이는 앱 열기 또는 문자 인식 설정을 먼저 누릅니다.

첫 화면은 `위챗 백업`, `카카오톡 백업`, `결과 보기` 세 선택지를 먼저 보여줍니다. 브라우저가 화면을 준비하는 중에도 같은 첫 화면에 `여러 방을 한 번에 저장`, `내 컴퓨터에 저장`, `외부 전송 기본 꺼짐`, `입력할 내용 없음` 안내가 먼저 보입니다. 여러 방을 한 번에 저장할 때는 같은 첫 화면의 `위챗 통째 백업 확인` 또는 `카카오톡 통째 백업 확인`을 누릅니다. 앱 상태나 문자 인식 확인은 막힐 때 펼쳐 보는 도움말에 들어 있습니다. 개인 PC에서 더블클릭으로 쓰는 기본 흐름은 추가 입력 없이 진행합니다.
위챗 백업, 카카오톡 백업, 결과 보기는 모두 같은 웹 화면 안에서 이어집니다.

## 요구사항

- Windows PC
- 카카오톡 또는 위챗 Windows 앱
- 카카오톡 백업: Windows 한국어 문자 인식
- 위챗 백업: Windows 중국어 문자 인식. Windows 언어 추가에서 `중국어`를 검색하고 `중국어(간체, 중국)` 또는 `중국어(번체, 대만/홍콩)` 중 하나를 기본 선택 그대로 설치하면 됩니다.
- 백업 화면 실행 도구는 없으면 `1_백업_시작.bat` 또는 `시작하기.bat`가 자동 설치를 시도합니다

카카오톡이나 위챗 앱이 설치되어 있지 않으면 시작 화면, 백업 화면, 준비 확인 화면의 `공식 설치 페이지` 버튼을 눌러 설치합니다. 설치와 로그인이 끝나면 웹 화면의 `상태 새로고침`을 누릅니다.

설치 창이 뜨면 허용하고 기다리면 됩니다. Windows가 실행 확인 창을 띄우면 GitHub Releases에서 받은 공개 ZIP인지 확인한 뒤 `실행` 또는 `허용`을 누릅니다. 모르는 출처에서 받은 파일이면 실행하지 않습니다. 자동 설치가 막히면 압축을 푼 폴더에 생성되는 `4_실행도구_설치.url`을 더블클릭해 설치 페이지를 열고 LTS 버전을 설치합니다. 설치가 끝나면 브라우저와 검은 창을 닫고 `1_백업_시작.bat`를 다시 더블클릭합니다. Windows가 파일 확장자를 숨기면 `1_백업_시작`으로 보일 수 있습니다. 보이지 않으면 `시작하기` 또는 `시작하기.bat`를 사용합니다.

명령어를 찾을 필요는 없습니다. 설치가 계속 막히면 압축을 푼 폴더의 `3_문제_확인.txt` 맨 위의 빠른 해결 3단계를 먼저 봅니다. 이 파일은 처음부터 들어 있고, 백업 화면을 실행하면 문제 확인 기록도 자동으로 최신 내용으로 갱신됩니다.

## 설치

1. 공개 배포 Releases(<https://github.com/GSK-KR/computer-use-public/releases>)에서 `computer-use-public.zip` 공개 릴리스 압축 파일을 받습니다.
2. 원하는 폴더에 압축을 풉니다. ZIP 파일 안에서 바로 실행하지 않습니다.
3. 압축을 푼 폴더에서 `scripts`와 `web` 폴더가 같이 보이는지 확인합니다.
4. `00_처음_여기부터.txt`를 열어 짧은 순서를 확인하거나, 바로 `1_백업_시작.bat`를 더블클릭합니다. Windows가 파일 확장자를 숨기면 `1_백업_시작`으로 보일 수 있습니다.

압축을 푼 폴더에 파일이 여러 개 보여도 처음 볼 것은 두 개뿐입니다. 안내는 `00_처음_여기부터.txt`, 실행은 `1_백업_시작.bat`입니다. `4_준비_보고서.bat`와 `4_검증_보고서.bat`는 지원용 보고서를 만들 때만 사용하며, 처음 백업 시작 파일이 아닙니다. 백업 전 준비 상태만 보내야 하면 첫 화면이나 `준비 확인`의 `준비 보고서`를 누릅니다. 브라우저를 열기 어려운 상태에서 준비 상태만 보내야 하면 `4_준비_보고서.bat`를 실행합니다. 백업 뒤 실사용 검증 보고서가 필요하면 첫 화면의 `막힐 때만 자세히 보기` 안에 있는 `위챗 찾기 / 막힐 때` 영역이나 `진행 기록`에서 `검증 보고서`를 누르고, 브라우저를 찾기 어려울 때만 `4_검증_보고서.bat`를 실행합니다. 나중에 정리할 때만 `5_정리_삭제.bat`를 사용합니다. 나머지 파일과 폴더는 자동 실행과 참고 문서입니다.

`필수 실행 파일이 빠졌습니다`가 보이면 압축이 제대로 풀리지 않았거나 일부 파일만 옮긴 상태이므로 폴더 전체를 다시 압축 해제한 뒤 실행합니다. `백업 저장 폴더를 준비하지 못했습니다` 또는 저장할 수 없다는 안내가 나오면 압축을 푼 폴더를 바탕화면이나 문서처럼 내가 저장할 수 있는 일반 폴더로 옮긴 뒤 `1_백업_시작.bat`를 다시 실행합니다. OneDrive 동기화 오류가 난 폴더, 회사 보안 폴더, 읽기 전용 폴더는 피합니다.

GitHub의 `Code > Download ZIP`으로 받는 소스 ZIP은 개발자용입니다. 일반 사용자는 `computer-use-public.zip` 공개 릴리스 압축 파일을 사용합니다.

압축을 푼 폴더에 `scripts`, `web`, `docs` 폴더가 보여도 처음 사용하는 사람은 열 필요가 없습니다. `scripts`와 `web`은 백업 화면 실행에 필요한 프로그램 폴더이고, `docs`는 같은 안내를 보관한 참고 폴더입니다.

## 실행

Windows에서 더블클릭합니다.

```text
1_백업_시작.bat
```

첫 실행에는 필요한 설정, 백업 결과 폴더, 진행 기록 폴더를 만들고 브라우저를 엽니다. 이후에는 같은 BAT를 다시 더블클릭하면 마지막 백업 화면을 엽니다. Windows가 파일 확장자를 숨기면 `1_백업_시작`으로 보일 수 있습니다. `시작하기.bat`는 같은 백업 화면을 여는 호환용 이름입니다.
검은 창에는 `카카오톡/위챗 백업 화면을 준비합니다`, `브라우저가 열릴 때까지 잠시 기다리세요`, `브라우저가 열리면 위챗 백업 또는 카카오톡 백업을 누르세요`, `여러 방을 한 번에 저장하려면 첫 화면의 위챗 통째 백업 확인 또는 카카오톡 통째 백업 확인을 누르세요` 문구가 보입니다. 창이 잠시 남아 있어도 기다리면 됩니다. 브라우저가 안 열리면 새로 만들어지는 `2_백업_화면.url`, 위챗 직접 진입은 `2_1_위챗_백업.url`, 카카오톡 직접 진입은 `2_2_카카오톡_백업.url`, 통째 백업 직접 진입은 `2_3_위챗_통째백업.url` 또는 `2_4_카카오톡_통째백업.url`, 계속 막히면 `3_문제_확인.txt` 맨 위의 빠른 해결 3단계를 먼저 봅니다.

브라우저가 자동으로 열리지 않으면 압축을 푼 폴더에 이번 실행에서 새로 생성된 `2_백업_화면.url`을 더블클릭합니다. 보이지 않으면 `백업 화면.url`을 더블클릭합니다. 위챗만 바로 열려면 `2_1_위챗_백업.url`, 카카오톡만 바로 열려면 `2_2_카카오톡_백업.url`을 더블클릭합니다. 여러 방을 한 번에 저장하려면 `2_3_위챗_통째백업.url` 또는 `2_4_카카오톡_통째백업.url`을 더블클릭합니다. 그래도 막히면 `3_문제_확인.txt` 맨 위의 빠른 해결 3단계만 먼저 따라 합니다.

주소 숫자는 PC마다 달라질 수 있고 정상입니다. 일반 사용자는 아무 값도 찾거나 입력하지 않습니다. 따로 입력할 내용이 없습니다. 예전 주소가 보여도 화면이 자동으로 깨끗한 주소로 바뀝니다. 오래된 화면이나 영어 접속 오류가 보이면 값을 찾지 말고 브라우저와 검은 창을 닫은 뒤 `1_백업_시작.bat`를 다시 실행합니다. 검은 창에 `백업 화면을 열지 못했습니다`가 보이면 같은 순서로 다시 열고, 계속 막히면 압축을 푼 폴더의 `3_문제_확인.txt` 맨 위의 빠른 해결 3단계를 먼저 봅니다.

위챗이 막히면 시작 화면의 `막힐 때만 자세히 보기`를 펼치고 `위챗 백업으로 이동`, `상태 새로고침`, `준비 확인`, `준비 보고서`, `진행 기록`, `검증 보고서`, `백업 폴더 열기`를 확인합니다.

## 백업 순서

1. 시작 화면의 `지금 할 일`을 먼저 봅니다. 위챗이 준비됐으면 `위챗 백업`, 카카오톡만 준비됐으면 `카카오톡 백업`, 준비가 막혔으면 `중국어 문자 인식 설치`, `한국어 문자 인식 설치`, `위챗 열기`가 먼저 보입니다.
2. 시작 화면에서 `위챗 열기` 또는 `카카오톡 열기`를 누르고 앱에 로그인합니다.
   앱이 없으면 같은 화면의 `공식 설치 페이지` 버튼을 누르고 설치와 로그인을 마친 뒤 `상태 새로고침`을 누릅니다.
3. 앱에서 백업할 방을 선택합니다.
   백업이 시작되면 끝날 때까지 앱 창을 최소화하거나 다른 창으로 가리지 않습니다.
4. 브라우저로 돌아와 `위챗 백업` 또는 `카카오톡 백업`을 누릅니다.
5. 백업 화면의 `3분 첫 백업` 안내에서 `앱 열기` -> `방 선택` -> `선택 완료 표시` -> `백업 버튼` 순서를 확인합니다. 준비가 막혀 있으면 이 영역의 첫 버튼도 `중국어 문자 인식 설치`, `한국어 문자 인식 설치`, `위챗 열기`처럼 먼저 해야 할 일로 바뀝니다.
6. 처음이면 `지금 열린 위챗 방 백업` 또는 `카카오톡 열린 방 백업`을 선택합니다. 준비가 막혀 있으면 추천 카드가 먼저 누를 버튼을 알려주고 바로 실행합니다.
7. 화면의 `방 선택 완료, 앱 창 앞에 둠` 표시는 체크박스입니다. 앱에서 방을 선택하고 앱 창을 앞에 둔 상태인지 확인한 뒤 체크하고 `백업`을 누릅니다. 체크를 빼먹고 `백업`을 눌러도 `방 선택 완료 표시가 필요합니다` 카드가 나오며, 그 카드의 `방 선택 완료 표시로 이동`을 누르면 체크 위치로 돌아가고 잠깐 강조됩니다.
8. 여러 방을 한 번에 저장하려면 첫 화면의 `위챗 통째 백업 확인` 또는 `카카오톡 통째 백업 확인`을 누릅니다. 백업 화면에서 앱 창을 앞에 두고 왼쪽 채팅 목록이 보이게 한 뒤 `전체 목록 확인 준비 완료` 체크박스를 선택하고 `전체 목록 확인`을 누르면 후보가 먼저 보입니다. 후보가 맞을 때 `목록 백업 실행`으로 진행합니다. 앱이나 문자 인식이 막혀 있으면 화면에 보이는 앱 열기 또는 문자 인식 설정을 먼저 누릅니다.
9. 끝나면 `결과 보기`에서 대화를 확인합니다. 필요한 방을 선택한 뒤 `표 파일 저장` 또는 `텍스트 저장`을 누르면 지금 보고 있는 방만 따로 저장할 수 있습니다. 현재 왼쪽 목록 전체는 `전체 표 파일 저장` 또는 `전체 텍스트 저장`으로 한 번에 저장할 수 있습니다. 결과 화면에는 `전체 저장은 현재 왼쪽 목록 기준입니다` 안내가 보이고, 검색이나 필터를 바꾸면 전체 저장 범위도 같이 바뀝니다.

카카오톡/위챗 `통째 백업(왼쪽 목록 전체 순회)`은 먼저 `전체 목록 확인`으로 번호가 붙은 후보 목록을 본 뒤, 완료 카드의 후보 방 카드가 맞으면 `목록 백업 실행`을 누릅니다. `목록 백업 실행`을 먼저 눌러도 바로 클릭을 시작하지 않고 `먼저 전체 목록 확인이 필요합니다` 카드가 나오며, 그 카드의 `전체 목록 확인`을 누르면 됩니다. 후보를 찾지 못했다는 카드가 나오면 목록 백업을 실행하지 말고 카드의 `카카오톡 열기` 또는 `위챗 열기`로 앱을 앞으로 가져온 뒤 왼쪽 채팅 목록이 보이게 맞추고 `목록 확인 다시`를 누릅니다. 이 기능은 왼쪽 목록을 끝까지 스크롤하고 새 방이 더 이상 나오지 않으면 자동으로 멈춥니다. 완료 카드에 `왼쪽 목록 끝까지 백업했습니다`가 보이면 새 방이 더 이상 나오지 않아 목록 끝까지 확인한 상태입니다. 카카오톡 목록 백업은 방마다 기본 120장까지 과거 대화를 읽고, 캡처 수 한도에 닿은 방은 결과 보기에서 `이력 확인`으로 표시됩니다. 방 하나가 일시적으로 열리지 않거나 방별 백업이 실패하면 기본 1회 자동으로 다시 시도하고, 그래도 안 되면 해당 방을 `확인 필요`로 남깁니다. 앱 화면을 읽어 순회하는 백업이고, 카카오톡이나 위챗 내부 DB를 직접 여는 방식은 아닙니다. 카카오톡 `목록에서 찾기`와 `오픈채팅`은 방 이름이나 오픈채팅 제목을 입력한 뒤 실행합니다.
`방 개수 상한` 또는 `페이지 상한`에 도달했다는 안내가 보이면 전체 목록이 끝났다고 보장하기 어렵습니다. 전체 목록 확인 단계라면 아직 방을 클릭하지 않은 상태이므로 완료 카드의 `상한 늘려 전체 목록 다시 확인`을 눌러 더 큰 상한으로 후보 확인부터 다시 진행합니다. 더 올릴 수 없는 상태에서 현재 후보만 저장해도 되면 같은 카드의 `현재 후보로 목록 백업 실행`을 누릅니다. 목록 백업 실행 뒤라면 저장된 결과는 먼저 확인할 수 있고, 같은 버튼으로 더 큰 상한의 `전체 목록 확인`부터 다시 시작합니다. 가능한 상한을 이미 최대로 올린 상태면 같은 버튼을 반복하지 않고 `저장된 결과 보기`와 `확인 필요 방 보기`로 빠진 방을 본 뒤 다시 백업합니다.
`일부 방은 확인이 필요합니다`가 보이면 백업 전체가 사라진 것이 아닙니다. 먼저 `저장된 결과 보기`로 저장된 방을 확인하고, 백업 화면의 확인 필요 방 카드에서 빠진 방을 봅니다. 더 자세한 내용은 완료 카드의 `확인 필요 방 보기`나 진행 로그의 `확인 필요 방` 목록에서 확인한 뒤 같은 백업을 다시 실행합니다.

결과 화면이 좁으면 새 주소를 열지 말고 같은 화면의 `결과 영역 크게 보기`를 누릅니다. 결과 화면에서 `정상`은 읽을 수 있는 백업이고, `확인 필요`는 실패가 아니라 발신자, 글자, 원본만 대조하면 된다는 뜻입니다. 방을 선택한 상태에서는 `표 파일 저장` 또는 `텍스트 저장`으로 현재 방만 따로 저장할 수 있습니다. 현재 왼쪽 목록 전체는 `전체 표 파일 저장` 또는 `전체 텍스트 저장`으로 한 번에 저장할 수 있습니다. 전체 저장 범위는 현재 왼쪽 목록 기준이고, 검색이나 필터를 바꾸면 저장 범위도 같이 바뀝니다. 결과 화면이 비어 있으면 실패가 아니라 아직 이 화면에서 읽을 백업 파일이 없는 상태일 수 있습니다. `위챗 백업` 또는 `카카오톡 백업`을 먼저 실행하고 `결과 새로고침`, `백업 폴더 열기`, `진행 기록` 순서로 확인합니다. 백업 화면에 `백업은 끝났지만 결과를 아직 찾지 못했습니다` 카드가 보이거나 결과가 계속 비어 있으면 `같은 백업 다시 실행`, `백업 폴더 열기`, `진행 기록`을 확인합니다.

## 문자 인식 설정

문자 인식이 준비되지 않았다고 나오면 시작 화면이나 백업 카드의 `중국어 문자 인식 설치` 또는 `한국어 문자 인식 설치`를 먼저 누릅니다. Windows 권한 확인 창이 뜨면 `예`를 누릅니다. 창이 보이지 않으면 작업 표시줄의 새 권한 확인 창 또는 방패 아이콘을 확인합니다. 자동 설치가 막히면 준비 확인 화면의 `Windows 언어 설정 열기`를 누릅니다.

1. 열린 Windows 설정에서 `언어 추가`를 누릅니다.
2. 카카오톡만 쓰면 `한국어`, 위챗을 쓰면 `중국어`를 검색한 뒤 `중국어(간체, 중국)` 또는 `중국어(번체, 대만/홍콩)` 중 하나를 추가합니다.
3. 설치할 기능을 묻는 화면이 나오면 기본 선택 그대로 설치합니다. 문자 인식 기능은 선택 해제하지 않습니다. Windows 표시 언어와 키보드는 바꾸지 않아도 됩니다.
4. 설치가 끝날 때까지 기다린 뒤 웹 화면으로 돌아와 `상태 새로고침`을 누릅니다. 바로 바뀌지 않으면 1~2분 뒤 한 번 더 누릅니다. 영어와 한국어만 설치된 PC는 위챗 백업 준비가 끝난 것으로 보지 않습니다.

카카오톡과 위챗 중 한쪽만 먼저 준비된 상태는 실패가 아닙니다. 시작 화면에 `준비할 항목 있음`이 보이면 준비된 앱부터 백업할 수 있고, 나머지는 화면에 보이는 `중국어 문자 인식 설치`, `한국어 문자 인식 설치`, `위챗 열기`, `공식 설치 페이지` 같은 버튼만 누르면 됩니다.

처음 백업은 Windows 문자 인식으로 시작합니다. 이유는 대화 화면의 위치와 목록 후보를 내 PC에서 바로 읽어야 하고, 기본 백업 결과를 외부로 보내지 않기 위해서입니다. AI 보정은 글자 품질을 더 높이고 싶을 때 쓰는 선택 기능이며, 외부 전송 가능성이 있으므로 기본으로 켜지지 않습니다.

## 화면 안내

- `시작`: `위챗 백업`, `카카오톡 백업`, `결과 보기`, `위챗 통째 백업 확인`, `카카오톡 통째 백업 확인`, `지금 할 일`, 막힐 때 확인 버튼. 문자 인식 설정, 방 선택, 결과 확인 순서는 막힐 때 펼쳐 보는 도움말에 있습니다.
- `위챗 백업`: 지금 열린 위챗 방과 `통째 백업(왼쪽 목록 전체 순회)`
- `카카오톡 백업`: 열린 채팅방, `통째 백업(왼쪽 목록 전체 순회)`, 목록에서 찾기, 오픈채팅 백업
- `결과 보기`: 카카오톡/위챗 백업 결과 확인, `위챗 결과`로 위챗만 보기, 방을 선택한 뒤 `표 파일 저장` 또는 `텍스트 저장`, 현재 왼쪽 목록 전체를 `전체 표 파일 저장` 또는 `전체 텍스트 저장`, 전체 저장 범위 안내, 같은 화면 안에서 결과 영역 크게 보기, 위챗 백업으로 돌아가기. 내부 파일에 남는 `Unknown`, `screenshot OCR` 같은 표현은 화면에서는 `발신자 확인`, `원본 캡처`, `문자 인식`처럼 다음 행동 기준으로 보입니다.
- `준비 확인`: 막힌 항목과 원클릭 해결 버튼. 지원 요청 전에는 `준비 보고서`로 채팅 내용 없이 앱, 문자 인식, 저장 폴더 상태만 저장할 수 있습니다.
- `진행 기록`: 실행한 백업의 진행 상태와 기록. 백업 전 막힘 상태만 보내야 하면 `준비 보고서`, 백업 뒤 실사용 확인이 필요하면 첫 화면이나 진행 기록의 `검증 보고서`를 눌러 결과 개수와 준비 상태만 담은 보고서를 만듭니다. 검증 보고서는 브라우저 준비 중 첫 화면에도 `위챗 백업`, `카카오톡 백업`, `결과 보기`, `통째 백업 확인`, `외부 전송 기본 꺼짐`, `입력할 내용 없음` 안내가 보이는지 같이 확인합니다. 보고서에 `요청 시간이 초과됐습니다`가 보이면 브라우저와 검은 창을 닫고 `1_백업_시작.bat`를 다시 실행합니다. 지원 요청이 필요할 때만 접힌 지원 영역을 펼치고, 먼저 `보기용 기록 저장`을 사용합니다. `원본 지원 묶음 저장`은 일부 대화 내용, 오류 원문, 폴더 경로가 포함될 수 있으므로 담당자가 요청했을 때만 확인란을 선택한 뒤 사용합니다.
- `저장 위치`: 평소에는 백업 저장 폴더 하나만 확인하면 됩니다. 백업 폴더 열기, 결과 보기, 진행 기록으로 바로 이동할 수 있고, 백업 폴더를 바꿀 때는 `C:\` 또는 `D:\`처럼 드라이브 문자로 시작하는 Windows 폴더를 입력합니다. 저장 권한 문제를 고친 뒤에는 `1_백업_시작.bat`를 다시 실행합니다.

`저장 위치` 화면은 기본 로컬 실행에서 실제 Windows 백업 폴더를 보여줍니다. 예를 들어 `C:\Users\내이름\computer-use\shots`처럼 보이고, `백업 폴더 열기`를 누르면 Explorer에서 바로 열립니다.

고급 옵션은 필요할 때만 펼칩니다. 처음에는 기본값 그대로 시작하는 것이 가장 안전합니다. 방 하나만 반복해서 실패하면 `방 재시도 횟수`를 2~3 정도로 올린 뒤 `전체 목록 확인`부터 다시 실행합니다.

## 삭제

나중에 이 도구를 치우고 싶으면 압축을 푼 폴더의 `5_정리_삭제.bat`를 더블클릭합니다. 바탕화면 실행 파일, 주소 바로가기, 문제 확인 파일, 준비/검증 보고서, 로컬 실행 설정만 정리하고, 백업 결과와 진행 기록은 지우지 않습니다.

```text
5_정리_삭제.bat
```

완전히 지우는 작업은 실수하면 복구하기 어렵습니다. 백업 산출물까지 지우려면 필요한 백업 파일을 먼저 다른 곳에 보관한 뒤 고급 삭제 방법을 확인합니다.

## 안전 원칙

- 본인이 접근 권한을 가진 카카오톡/위챗 대화만 백업합니다.
- 백업 결과, 진행 기록, 실행 설정에는 개인 데이터가 들어갈 수 있으므로 압축을 푼 폴더를 그대로 공유하지 않습니다.
- 쓰기, 삭제 같은 위험 작업은 화면의 확인 단계를 거칩니다.
- 기본 백업 화면은 이 컴퓨터에서만 열리도록 동작합니다.
- 나중에 MCP나 외부 공개 기능을 붙이더라도 지금 로컬 화면을 그대로 인터넷에 열지 않고, 별도 계정, 권한, HTTPS, 감사 로그가 필요합니다.
