# 카카오톡/위챗 백업 빠른 시작

Windows에서 카카오톡과 위챗 대화를 내 컴퓨터에 저장하고 같은 웹 화면에서 확인하는 도구입니다. 처음 목표는 방 하나를 백업하고 `결과 보기`에서 확인하는 것입니다.

처음 볼 것은 두 개뿐입니다.

- 안내: `00_처음_여기부터.txt`
- 실행: `1_백업_시작.bat`

Windows가 파일 확장자를 숨기면 `1_백업_시작`으로 보일 수 있습니다. `시작하기.bat`도 같은 화면을 여는 보조 실행 파일입니다. 기본 사용은 더블클릭과 한국어 웹 화면의 버튼만으로 진행하며, 주소 뒤의 값이나 접속 코드를 입력하지 않습니다.

## 요구사항

- Windows PC
- 카카오톡 또는 위챗 Windows 앱과 로그인
- 카카오톡 백업: Windows 한국어 문자 인식
- 위챗 백업: Windows 중국어 문자 인식

WSL은 없어도 됩니다. 예전 백업 파일 검사나 추가 검수 같은 선택 기능에서만 사용할 수 있으며, 기본 설치와 카카오톡/위챗 백업은 WSL 점검 때문에 멈추지 않습니다.

## 설치

1. [공개 배포 Releases](https://github.com/GSK-KR/computer-use-public/releases)에서 `computer-use-public.zip`을 받습니다. GitHub의 `Code > Download ZIP` 소스 압축은 개발자용입니다.
2. ZIP 파일을 원하는 폴더에 완전히 압축 해제합니다. ZIP 안에서 바로 실행하지 않습니다.
3. 압축을 푼 폴더에 `scripts`와 `web` 폴더가 함께 있는지 확인합니다.
4. `1_백업_시작.bat`를 더블클릭합니다.

첫 실행에는 필요한 실행 도구가 없으면 자동 설치를 시도하고, 백업 폴더와 진행 기록 폴더를 만든 뒤 브라우저를 엽니다. Windows가 실행 확인 창을 띄우면 GitHub Releases에서 받은 공개 ZIP인지 확인한 뒤 `실행` 또는 `허용`을 누릅니다. 모르는 출처에서 받은 파일이면 실행하지 않습니다.

자동 설치가 막히면 압축을 푼 폴더에 생기는 `4_실행도구_설치.url`을 더블클릭해 설치 페이지를 열고 LTS 버전을 설치합니다. 설치 후 브라우저와 검은 창을 닫고 `1_백업_시작.bat`를 다시 실행합니다.

`필수 실행 파일이 빠졌습니다`가 보이면 일부 파일만 옮긴 상태입니다. 폴더 전체를 다시 압축 해제합니다. `백업 저장 폴더를 준비하지 못했습니다`가 보이면 압축을 푼 폴더를 바탕화면이나 문서처럼 쓸 수 있는 일반 폴더로 옮깁니다. OneDrive 오류 폴더, 회사 보안 폴더, 읽기 전용 폴더는 피합니다.

## 실행

평소에는 매번 `1_백업_시작.bat`를 더블클릭하면 됩니다. 설정, 서버 시작, 브라우저 열기까지 자동으로 처리합니다.

브라우저가 자동으로 열리지 않으면 이번 실행에서 새로 만들어진 `2_백업_화면.url`을 더블클릭합니다. 바로가기 파일은 다음과 같습니다.

- `2_1_위챗_백업.url`: 위챗 백업
- `2_2_카카오톡_백업.url`: 카카오톡 백업
- `2_3_위챗_통째백업.url`: 위챗 통째 백업
- `2_4_카카오톡_통째백업.url`: 카카오톡 통째 백업

주소 번호는 PC마다 달라질 수 있습니다. 외우거나 직접 입력할 필요가 없습니다. 위챗 백업, 카카오톡 백업, 결과 보기는 모두 같은 웹 화면 안에서 이어집니다.

첫 화면에는 필요한 선택만 보입니다.

- `위챗 백업`
- `카카오톡 백업`
- `결과 보기`
- `위챗 통째 백업`
- `카카오톡 통째 백업`

앱 실행이나 문자 인식이 준비되지 않았을 때만 `준비가 안 될 때`를 펼칩니다. 여기서 앱 열기, 공식 설치 페이지, 문자 인식 설치, 준비 확인을 사용할 수 있습니다.

## 백업 순서

### 방 하나 백업

1. 카카오톡 또는 위챗 앱을 열고 로그인합니다.
2. 앱에서 백업할 방을 열고 앱 창을 앞에 둡니다.
3. 브라우저에서 해당 앱의 백업 메뉴를 누릅니다.
4. `방 선택 완료, 앱 창 앞에 둠` 체크박스를 선택합니다.
5. `지금 열린 방 백업`을 누릅니다.
6. 백업이 끝날 때까지 앱 창을 최소화하거나 다른 창으로 가리지 않습니다.
7. `결과 보기`에서 저장된 대화를 확인합니다.

체크를 빼먹으면 백업을 시작하지 않고 `방 선택 완료 표시가 필요합니다`를 보여 줍니다. 안내 버튼을 누르면 체크 위치로 돌아갑니다.

카카오톡 채팅 창이 여러 개 열려 있어도 `지금 열린 방 백업`은 지금 앞에 둔 창 하나만 저장합니다. 다른 채팅 창은 닫거나 건드리지 않습니다. `목록에서 찾기`는 왼쪽 목록의 방 제목을 찾아 열고, 도구가 새로 연 창만 백업 뒤 닫습니다.

### 모든 방 통째 백업

1. 첫 화면에서 `위챗 통째 백업` 또는 `카카오톡 통째 백업`을 누릅니다.
2. 앱 창을 앞에 두고 왼쪽 채팅 목록이 보이게 합니다.
3. `전체 목록 확인 준비 완료` 체크박스를 선택합니다.
4. `1. 전체 목록 확인`을 누릅니다. 이 단계에서는 개별 채팅방을 열지 않고 후보 이름만 보여 줍니다. 카카오톡의 `조용한 채팅방` 보관함이 있으면 안의 후보를 세기 위해 보관함 목록만 열었다가 닫습니다.
5. 후보가 맞으면 `2. 확인한 목록 백업`을 누릅니다.
6. 완료 후 `결과 보기`에서 저장된 방을 확인합니다.

통째 백업은 왼쪽 목록을 계속 스크롤하고 새 후보가 없는 화면을 연속 4번 확인하면 자동으로 멈춥니다. 기본 상한은 500페이지와 2,000개 방이며, 글자를 조금 다르게 읽은 겹친 행은 다시 세지 않고 이름이 같은 서로 다른 행은 각각 처리합니다. 카카오톡의 `조용한 채팅방` 보관함도 함께 순회합니다. 완료 카드에 `왼쪽 목록 끝까지 백업했습니다`가 보이면 일반 목록과 보관함의 끝까지 확인한 상태입니다. 방 하나가 일시적으로 실패하면 기본 1회 다시 시도하고, 그래도 실패하면 `확인 필요 방 보기`에 남깁니다.

`확인한 목록 백업`을 먼저 눌러도 방을 바로 클릭하지 않습니다. `먼저 전체 목록 확인이 필요합니다` 안내에서 후보 확인부터 시작합니다. 후보가 없으면 앱의 왼쪽 채팅 목록을 다시 보이게 한 뒤 `목록 확인 다시`를 누릅니다.

`방 개수 상한` 또는 `페이지 상한`이 보이면 전체 목록이 끝났다고 보장할 수 없습니다. `상한 늘려 전체 목록 다시 확인`으로 후보 확인부터 다시 진행합니다. 더 올릴 수 없고 현재 후보만 저장해도 되면 `현재 후보 백업`을 누릅니다.

카카오톡 통째 백업은 방마다 최대 500장, 위챗 통째 백업은 최대 800장까지 읽습니다. 각 방에서 최신 위치와 가장 오래된 위치를 연속 확인하고, 가장 오래된 위치에 먼저 닿으면 상한 전에도 끝납니다. 상한에 먼저 닿거나 맨 위·맨 아래 도달을 확인하지 못한 방은 결과 보기에서 `이력 확인`으로 표시됩니다. 사진·파일 카드는 화면 캡처와 보이는 정보가 저장되지만 원본 첨부파일 자체의 자동 다운로드는 보장하지 않습니다. 이 기능은 로그인된 앱 화면을 순회하며 내부 DB를 직접 여는 방식이 아닙니다.

## 결과 확인과 저장

`결과 보기`는 같은 웹 화면 안에서 열립니다.

- 방 하나 저장: 방을 선택하고 `표 파일 저장` 또는 `텍스트 저장`
- 현재 목록 전체 저장: `전체 표 파일 저장` 또는 `전체 텍스트 저장`
- 넓게 보기: `결과 영역 크게 보기`

전체 저장 범위는 현재 왼쪽 목록 기준입니다. 검색이나 필터를 바꾸면 저장 범위도 바뀝니다. `정상`은 읽을 수 있는 백업이고, `확인 필요`는 실패가 아니라 발신자, 글자, 원본을 대조할 항목이 있다는 뜻입니다.

결과가 비어 있으면 `결과 새로고침`을 누릅니다. 계속 비어 있으면 `같은 백업 다시 실행`, `백업 폴더 열기`, `진행 기록` 순서로 확인합니다. `저장 위치`에서는 실제 Windows 백업 폴더를 확인하고 Explorer로 열 수 있습니다.

## 문자 인식 설정

처음 백업은 Windows 문자 인식으로 시작합니다. 대화 화면과 목록을 내 PC에서 바로 읽고 기본 백업 결과를 외부로 보내지 않기 위해서입니다. AI 보정은 글자 품질을 더 높이고 싶을 때 사용하는 선택 기능이며 기본으로 켜지지 않습니다.

문자 인식 준비가 필요하면 `준비가 안 될 때` 또는 `준비 확인`에서 다음 버튼을 누릅니다.

- 카카오톡: `한국어 문자 인식 설치`
- 위챗: `중국어 문자 인식 설치`

Windows 권한 확인 창이 뜨면 `예`를 누릅니다. 자동 설치가 막히면 `Windows 언어 설정 열기`를 누르고 다음 순서로 진행합니다.

1. `언어 추가`를 누릅니다.
2. 카카오톡은 `한국어`를 추가합니다.
3. 위챗은 `중국어(간체, 중국)` 또는 `중국어(번체, 대만/홍콩)` 중 하나를 추가합니다.
4. 기본 선택 그대로 설치하고 문자 인식 기능은 해제하지 않습니다. Windows 표시 언어와 키보드는 바꾸지 않아도 됩니다.
5. 설치 후 웹 화면에서 `상태 새로고침`을 누릅니다. 바로 바뀌지 않으면 1~2분 뒤 다시 누릅니다.

카카오톡과 위챗 중 한쪽만 준비된 상태는 실패가 아닙니다. 준비된 앱부터 백업할 수 있습니다.

## 문제 해결

대부분은 다음 순서로 해결됩니다.

1. 브라우저와 검은 창을 닫습니다.
2. `1_백업_시작.bat`를 다시 더블클릭합니다.
3. 브라우저가 안 열리면 새로 생긴 `2_백업_화면.url`을 엽니다.
4. 계속 막히면 `3_문제_확인.txt` 맨 위의 빠른 해결 순서를 봅니다.

`404 not found`, `missing or invalid local token`, 오래된 영어 오류, 주소 뒤 긴 값이 보여도 아무 값을 입력하지 않습니다. 오래된 탭이나 주소일 수 있으므로 `1_백업_시작.bat`를 다시 실행합니다. 서버는 정적 백업 경로와 결과 경로를 같은 웹 화면으로 복구하며, 정상 실행에서는 로컬 토큰 입력란을 사용하지 않습니다.

지원 요청을 받았을 때만 보고서를 만듭니다.

- 백업 전 준비 상태: 웹의 `준비 보고서` 또는 `4_준비_보고서.bat`
- 백업 후 실사용 상태: 웹의 `검증 보고서` 또는 `4_검증_보고서.bat`

보고서에는 채팅 본문, 방 이름, 원본 스크린샷 경로를 넣지 않습니다. 원본 지원 묶음은 개인정보가 포함될 수 있으므로 담당자가 요청했을 때만 사용합니다.

## 삭제

`5_정리_삭제.bat`를 더블클릭하면 바로가기, 로컬 실행 설정, 지원 보고서를 정리합니다. 백업 결과와 진행 기록은 지우지 않습니다.

## 안전 원칙

- 본인이 접근 권한을 가진 대화만 백업합니다.
- 백업 결과와 진행 기록에는 개인 데이터가 들어갈 수 있으므로 압축을 푼 폴더를 그대로 공유하지 않습니다.
- 기본 백업 결과는 이 컴퓨터에 저장되고 외부 검수/번역은 기본으로 꺼져 있습니다.
- 기본 웹 화면은 이 컴퓨터에서만 열리도록 동작합니다.
- 나중에 MCP, Cowork 또는 외부 공개 기능을 붙일 때는 로컬 화면을 그대로 인터넷에 열지 않고 별도 계정, 권한 범위, HTTPS, 승인, 감사 로그를 사용해야 합니다.

<summary>선택 기능: AI에게 이 폴더로 작업 맡기기</summary>

이 방법은 명령어를 직접 고르지 않고 Claude Code, Codex CLI, Gemini CLI 중 하나에게 `이 프로젝트를 사용해서 원하는 일을 끝내 달라`고 맡기고 싶은 사람을 위한 방법입니다. 세 도구를 모두 설치할 필요는 없습니다. 이미 사용하는 AI 도구 하나만 있으면 됩니다.

### 1. AI 도구 하나 준비

AI 작업 도구는 이 ZIP에 포함되어 있지 않습니다. 계정과 인터넷 연결이 필요하며 이용 중인 요금제에 따라 비용이 생길 수 있습니다. 설치하지 않았다면 원하는 도구 하나의 공식 안내만 따라 합니다.

- [Codex CLI 공식 안내](https://developers.openai.com/codex/cli/)
- [Claude Code 공식 빠른 시작](https://code.claude.com/docs/en/quickstart)
- [Gemini CLI 공식 안내](https://google-gemini.github.io/gemini-cli/)

설치가 어려우면 세 도구 중 평소 사용하는 서비스의 안내 하나만 선택합니다. 카카오톡·위챗 백업 자체는 AI 도구 없이도 `1_백업_시작.bat`로 사용할 수 있습니다.

### 2. 반드시 압축을 푼 프로젝트 폴더에서 시작

1. `computer-use-public.zip`을 완전히 압축 해제합니다.
2. 압축을 푼 폴더를 Windows 파일 탐색기로 엽니다.
3. 위쪽 주소 표시줄을 한 번 클릭하고 `powershell`을 입력한 뒤 Enter를 누릅니다.
4. 파란색 또는 검은색 PowerShell 창의 현재 위치가 방금 연 폴더인지 확인합니다.

`scripts` 폴더 안이나 ZIP 파일이 보이는 다운로드 폴더에서 시작하면 AI가 루트 지침을 찾지 못할 수 있습니다. `AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, `README.md`, `1_백업_시작.bat`가 함께 보이는 폴더가 시작 위치입니다.

### 3. 사용하는 AI 하나만 실행

Codex를 쓰는 경우:

```powershell
codex
```

Claude Code를 쓰는 경우:

```powershell
claude
```

Gemini CLI를 쓰는 경우:

```powershell
gemini
```

`명령을 찾을 수 없습니다`라는 오류가 나오면 선택한 AI 도구가 아직 설치되지 않은 것입니다. 위의 해당 공식 안내에서 설치와 로그인을 마친 뒤 같은 폴더에서 다시 실행합니다. 신뢰할 폴더인지 묻는 화면이 나오면 GitHub Releases에서 받은 ZIP을 직접 압축 해제한 폴더가 맞는지 확인한 뒤 진행합니다.

### 4. 첫 요청을 그대로 입력

AI 도구가 열리면 다음 문장을 그대로 입력해도 됩니다.

```text
이 폴더의 AGENTS.md와 docs/ai_agent_guide.md를 먼저 참고하고, 이 프로젝트를 사용해서 내가 요청하는 작업을 끝까지 실행한 뒤 결과까지 검증해 줘. 먼저 프로젝트 준비 상태를 확인해 줘.
```

정상이라면 AI가 `node scripts/ai_project_check.mjs --json`을 실행하고 패키지 상태를 확인합니다. AI가 루트 지침을 자동으로 읽더라도 위 문장을 한 번 명시하면 시작 위치가 잘못된 문제를 더 쉽게 발견할 수 있습니다.

그다음 원하는 일을 평소 말하듯 요청합니다.

```text
지금 열어 둔 위챗 방을 백업하고 결과가 정상인지 확인해 줘.
```

```text
카카오톡 왼쪽 목록의 모든 방을 먼저 미리 확인하고, 빠질 가능성이 있는 방을 알려 준 뒤 전체 백업해 줘.
```

```text
이 프로젝트의 Windows Chrome 자동화로 로그인된 사이트의 주문 목록을 읽어 표 파일로 정리해 줘. 사이트 내용은 바꾸지 마.
```

```text
이 프로젝트로 B, C, D 작업을 순서대로 처리하고 각 단계의 결과 파일과 검증 결과를 알려 줘.
```

### 5. 사용자가 직접 해야 하는 순간

AI가 대부분의 준비와 실행 명령을 고르지만 다음 단계는 사용자가 직접 해야 할 수 있습니다.

- 카카오톡·위챗 로그인 또는 QR 로그인
- 사이트 로그인, 문자 인증, MFA, CAPTCHA
- 같은 이름의 방이나 탭이 여러 개일 때 정확한 대상 선택
- 결제, 삭제, 공개 게시처럼 영향이 큰 마지막 확인
- Windows가 띄우는 앱 실행 또는 권한 허용 확인

AI가 로그인이나 CAPTCHA를 우회하도록 요구하지 않습니다. 로그인 후 `됐어, 계속해`라고 말하면 같은 세션에서 이어서 진행할 수 있습니다.

### 6. 개인정보 주의

AI 작업 도구는 프로젝트 파일을 읽고 필요한 명령을 실행할 수 있습니다. 백업 결과에는 대화 내용과 스크린샷이 들어갈 수 있으므로 `백업 전체를 외부로 올려 줘`, `모든 대화를 AI로 다시 읽어 줘`처럼 요청하기 전에는 사용하는 AI 서비스의 개인정보 정책과 전송 범위를 확인합니다.

이 프로젝트의 기본 백업은 로컬 Windows 문자 인식을 사용합니다. AI가 클라우드 문자 인식이나 번역을 제안하더라도 사용자가 명시적으로 동의하기 전에는 채팅 내용이나 스크린샷을 외부 서비스로 보내지 않도록 지침이 들어 있습니다.

AI가 읽는 공통 운영 규칙은 `AGENTS.md`, 자세한 작업별 명령과 완료 판정은 [AI 세션용 운영 안내](docs/ai_agent_guide.md)에 있습니다. Codex와 Gemini는 공용 `.agents/skills/run-computer-use` 스킬도 사용할 수 있고, Claude Code에는 같은 내용의 `.claude/skills/run-computer-use` 스킬이 들어 있습니다. 외부 가이드나 제작자의 개인 PC 경로는 필요하지 않습니다.

<summary>선택 기능: Windows Chrome 자동화</summary>

이 기능은 카카오톡/위챗 백업에는 필요하지 않습니다. 다른 웹사이트를 자동으로 읽거나 입력해야 할 때만 사용합니다.

Google Chrome과 Windows Node.js LTS가 설치되어 있으면 별도 실행 BAT나 외부 가이드 없이 이 공개 패키지만으로 사용할 수 있습니다. 첫 명령이 일반 Chrome과 분리된 자동화 전용 Chrome 창을 알아서 열며, WSL은 필요하지 않습니다.

```powershell
powershell -NoProfile -STA -ExecutionPolicy Bypass -File .\scripts\cu_web.ps1 -Action pages
powershell -NoProfile -STA -ExecutionPolicy Bypass -File .\scripts\cu_web.ps1 -Action goto -Arg1 https://example.com
powershell -NoProfile -STA -ExecutionPolicy Bypass -File .\scripts\cu_web.ps1 -Action read -Url example.com
```

처음 연결 모듈을 준비할 때만 인터넷 연결이 필요할 수 있습니다. 대상 탭 지정, 입력, 파일 업로드, 제출 전 검사, JSON 워크플로, 문제 해결, MCP/Claude/Cowork 연동 설계는 [Windows Chrome 웹 자동화 운영 안내](docs/browser_workflow_playbook.md)에 정리되어 있습니다.

</details>
